# ドラッグアンドドロップで並び替えできるようにするまで

作成日: 2026-05-06

## 実施したこと

1. todosテーブルに `order` カラムを追加
- マイグレーション `2026_05_06_000005_add_order_to_todos_table.php` を新規作成。
- `order` カラム（integer・デフォルト0）を追加。
- マイグレーション実行時に既存レコードへ `created_at DESC` の順で初期値を設定し、それまでの表示順を維持。

2. Todo モデルを更新
- `$fillable` に `order` を追加。

3. コントローラを更新
- `index` アクション：`latest()` から `orderBy('order')` に変更。
- `store` アクション：新しいTodoを先頭（`min(order) - 1`）に挿入するよう変更。
- `reorder` アクション（新規）：ID配列を受け取り各レコードの `order` を更新して `JsonResponse` を返す。

4. ルートを追加
- `PATCH /todos/reorder` → `todos.reorder` を追加。
- ルート衝突を避けるため `todos/reorder` を `todos/{todo}` より先に定義。

5. Blade レイアウトを更新（layouts/app.blade.php）
- CSRF metaタグを追加（JS から `X-CSRF-TOKEN` を取得するため）。
- SortableJS CDN を追加。
- `public/js/todos.js` の読み込みを追加。
- `@stack('scripts')` を配置（将来の拡張用）。

6. Todo一覧 Blade を更新（todos/index.blade.php）
- `<ul>` に `id="todo-list"` と `data-reorder-url="{{ route('todos.reorder') }}"` を追加。
- URLをBlade側で解決しJSに渡すことで、JSファイル内にテンプレート構文を書かずに済む構成にした。

7. Todo行パーシャルを更新（todos/partials/todo-item.blade.php）
- `<li>` に `data-id="{{ $todo->id }}"` を追加。
- ドラッグハンドル `<span class="drag-handle">` を先頭に追加。

8. JSを独立ファイルに切り出し（public/js/todos.js）
- 当初インラインスクリプト（`@push('scripts')`）として実装。
- `.js` ファイルへの切り出しに際して IIFE（即時実行関数式）が不要になるため除去。
- `function () {}` をアロー関数・省略メソッド記法に整理。
- reorderエンドポイントのURLは `list.dataset.reorderUrl` 経由で取得。

9. CSSを追加（public/css/todos.css）
- `.drag-handle`：`cursor: grab` など掴める見た目にするスタイル。
- `.todo-item-ghost`：ドラッグ中のプレースホルダー（半透明）スタイル。

- Todo一覧の `⠿` ハンドルをドラッグして順序を入れ替えられる。
- ドロップ時にページリロードなしで順序が DB に保存される（fetch PATCH）。
- 次回アクセス時もその順序で表示される（`orderBy('order')`）。
- 新しく追加したTodoは先頭に配置される。
