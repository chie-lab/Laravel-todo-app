# ユーザー認証（登録・ログイン）機能を追加するまで

作成日: 2026-05-06

## 実施したこと

1. AuthController を作成
- `showRegister` / `register` / `showLogin` / `login` / `logout` の5アクションを実装。
- 登録時は `User::create()` 後に `Auth::login()` で自動ログインし、Todoトップへリダイレクト。
- ログイン時は `Auth::attempt()` で認証。失敗した場合はメールアドレスを保持したままエラーメッセージを返す。
- ログアウト時はセッションを無効化・トークン再生成し、ログイン画面へリダイレクト。

2. ルートを再構成
- Todoの全ルートを `middleware('auth')` グループでまとめ、未ログインを `routes.php` レベルで遮断。
- `/register` / `/login` は `middleware('guest')` グループに配置し、ログイン済みユーザーは `todos.index` へリダイレクト。
- `POST /logout` は `middleware('auth')` 単体で追加。
- `todos/reorder` を `todos/{todo}` より前に定義してルート衝突を防ぐ構成を維持。

3. ログイン画面を作成（resources/views/auth/login.blade.php）
- `layouts.app` を継承。
- メールアドレス・パスワード入力フォーム。
- バリデーションエラーをアラートで表示。
- 新規登録画面へのリンクを末尾に配置。

4. 新規登録画面を作成（resources/views/auth/register.blade.php）
- `layouts.app` を継承。
- 名前・メールアドレス・パスワード・パスワード確認の4フィールド。
- バリデーションエラーをアラートで表示。
- ログイン画面へのリンクを末尾に配置。

5. 共通レイアウトを更新（resources/views/layouts/app.blade.php）
- `@auth` ディレクティブでログイン中のみヘッダーを表示。
- ヘッダーにログイン中のユーザー名とログアウトボタンを配置。

6. CSS を追加（public/css/todos.css）
- `.site-header` / `.site-header-user` / `.site-header-logout` / `.logout-button` でヘッダーを装飾。
- `.auth-card` / `.auth-form` / `.field` / `.btn-block` / `.auth-link` で認証フォームを装飾。

## 現在の到達点

- 未ログインで `/` にアクセスすると `/login` へリダイレクトされる。
- ログイン済みで `/login` `/register` にアクセスすると `/` へリダイレクトされる。
- ログインに成功すると Todo トップへ遷移し、画面右上にユーザー名とログアウトボタンが表示される。
- ログアウトすると `/login` へリダイレクトされる。
